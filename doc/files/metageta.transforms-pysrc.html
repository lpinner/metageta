 24  ==============

 27  Adding another stylesheet 
 28  ------------------------- 
 29      - Create a new/add an existing .xsl file that transforms crawler output (schema described below) to your XML schema. 
 30      - Create an xml file that describes your XML schema (see below) 
 31      - Save the xsl & xml file to the lib/transforms directory, it will be automatically available 
 32  Note: You may also pass the filepath to an external XSL stylesheet to the L{Transform} function. 
 33   
 34  Metadata schema 
 35  --------------- 
 36  The following schema defines a simple XML format which holds extracted metadata 
 37  (see L{DictToXML} and L{ListToXML}) to be transformed by an XSL stylesheet to 
 38  various XML metadata schemas:: 
 39      <?xml version="1.0" encoding="UTF-8" ?> 
 40      <xs:schema xmlns:xs="http://www.w3.org/2001/XMLSchema"> 
 41        <xs:element name="crawlresult"> 
 42          <xs:complexType> 
 43            <xs:sequence> 
 44              <xs:element ref="{FIELD}" /> 
 45            </xs:sequence> 
 46          </xs:complexType> 
 47        </xs:element> 
 48        <xs:element name="{FIELD}"> 
 49          <xs:complexType mixed="true" /> 
 50        </xs:element> 
 51      </xs:schema> 
 52  Where: {FIELD} are the fields defined in L{formats.fields}, plus any additional metadata elements you pass in. 
 53  For example, you can pass additional metadata elements in by adding them to the spreadsheet 
 54  which is passed to L{runtransform.py<runtransform>}. 
 55  Additional metadata elements for existing XSL stylesheets will be documented below. 
 56   
 57  Stylesheet description schema 
 58  ----------------------------- 
 59  The following schema defines an XML file that describes an XSL stylesheet to be used by this module 
 60  to transform from the simple XML format defined above to various XML metadata schemas, 
 61  e.g. the ANZLIC Metadata Profile:: 
 62      <?xml version="1.0" encoding="UTF-8" ?> 
 63      <xs:schema xmlns:xs="http://www.w3.org/2001/XMLSchema"> 
 64        <xs:element name="stylesheet"> 
 65          <xs:complexType> 
 66            <xs:attribute name="name" type="xs:string" use="required" /> 
 67            <xs:attribute name="file" type="xs:string" use="required" /> 
 68            <xs:attribute name="description" type="xs:string" use="required" /> 
 69          </xs:complexType> 
 70        </xs:element> 
 71      </xs:schema>
